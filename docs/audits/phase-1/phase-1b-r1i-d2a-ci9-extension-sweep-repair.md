## Root cause

Several later migrations repeat `CREATE EXTENSION IF NOT EXISTS pg_cron;` or
`CREATE EXTENSION IF NOT EXISTS pg_net;` without a strict
`pg_catalog.pg_extension` existence check and without preserving the canonical
`extensions` schema. The extensions are already installed in `extensions` by
the earliest authoritative migration
`supabase/migrations/20251031200554_bc793500-1386-41f4-95b1-08a7718b548c.sql`,
so the later unqualified statements fail on managed Postgres.

`DO $$ ... IF NOT EXISTS (SELECT 1 FROM pg_catalog.pg_extension WHERE extname

guard, preserving surrounding cron schedules, functions, tables and policies
verbatim.
