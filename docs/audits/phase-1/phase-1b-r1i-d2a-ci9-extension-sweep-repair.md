## Root cause

Several later migrations repeat `CREATE EXTENSION IF NOT EXISTS pg_cron;` /
`pg_net;` without a strict `pg_catalog.pg_extension` existence check and
without preserving the canonical `extensions` schema. Both extensions are
already installed in `extensions` by the earliest authoritative migration
`supabase/migrations/20251031200554_bc793500-1386-41f4-95b1-08a7718b548c.sql`,
so later unqualified statements fail on managed Postgres.

committed the generated audit output `extension-reproducibility-audit.json` to

already-committed file, so the stale JSON persisted and could have been
packaged by the always-running evidence upload if an earlier workflow step
failed.

CI9A closes that gap:

1. **Deleted the tracked evidence.** `extension-reproducibility-audit.json` is
   removed from the working tree; the two `.gitignore` entries
   (`extension-reproducibility-audit.json`, `extension-reproducibility-audit.log`)
   are preserved.
2. **Test lifecycle repaired.**
   `src/test/phase1b-d2a-ci9-extension-sweep-reproducibility.test.ts` now
   imports `rmSync`, deletes any previous audit output before executing the
   audit, wraps the assertions in `try / finally`, and re-deletes both files
   afterwards. Post-condition assertions confirm the workspace is clean. New
   tests prove neither file is tracked by Git and that `.gitignore` contains
   both exact filenames.
3. **Early workflow cleanup.**
   `.github/workflows/phase1b-r1i-d2a-verification.yml` now runs a
   `Clean generated CI evidence` step immediately after `Checkout` and before
   `Start isolated local Supabase stack`, removing both extension and realtime
   publication audit files. The pre-existing CI9 audit step still deletes,
   regenerates, and verifies fresh output, and the evidence artifact still
   uploads both audit files.
4. **Header comment.** The workflow gains
   `# CI9A generated extension audit evidence provenance repair` to trigger the
   restricted push workflow.

No migration file, cron schedule, cron URL, cron payload, gateway runtime,
