## 7. Schema-derived types

The project uses `src/integrations/supabase/types.ts`, regenerated by the
Lovable platform on migration acceptance. Because the canonical migration is
pending (not in `supabase/migrations/`), types are **intentionally not
regenerated in this slice**. Rationale:

- Regeneration requires the migration to be applied to the managed Test DB.
- Applying it is a production-adjacent action reserved for R1I-c.2 under
  Release Manager approval.
- No handler code is written in this slice, so no compile-time dependency on
  the new columns exists.

Drift prevention for c.2/c.3: the promotion procedure in the pending-migrations
README requires (a) hash-match on the promoted file and (b) types regeneration
+ full typecheck as part of the c.2 acceptance harness.
