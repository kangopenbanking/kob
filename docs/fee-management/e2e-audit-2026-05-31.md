# Fee Management — End-to-End Audit Report

**Date:** 2026-05-31  
**Scope:** `/admin/fee-management` — every fee structure exposed in the admin form  
**Auditor:** Lovable AI Agent  
**Result:** **PASS** (59/59 transaction types resolve to an active platform fee)

## 1. What was checked

For every transaction type selectable in `CreateFeeStructureForm` (the master
list driving the admin UI), we verified:

1. The type is permitted by the `fee_structures_transaction_type_check`
   database constraint.
2. At least one **active**, **platform-scope** row exists in
   `public.fee_structures` so the resolver in `useFeeEstimate.ts` and the
   server-side fee gates always return a concrete amount (no implicit fallback
   to hard-coded rates).
3. The newly added **Statement Download** fees flow through
   `resolve_statement_fee()` and are charged by `charge_statement_fee_v2()`
   against the user's `account_balances` with an idempotency key.

## 2. Gaps found and fixed

| # | Gap | Fix |
|---|-----|-----|
| 1 | `statement_download_consumer` / `statement_download_banking` were not part of the `fee_structures_transaction_type_check` constraint, so the unified Fee Management UI could not store them. | Constraint extended; both types added to `TRANSACTION_TYPES` in `CreateFeeStructureForm.tsx`. |
| 2 | No default platform fee for Statement Downloads — pricing was driven only by `statement_fee_settings`. | Seeded a `fixed = 500 XAF` platform row for both Consumers and Banking variants. |
| 3 | `resolve_statement_fee()` did not consult `fee_structures`, so admin edits in `/admin/fee-management` had no effect on statement pricing. | Updated the resolver to read `fee_structures` (platform scope, active, in date range) as priority **3** — between app/institution overrides and the global singleton. |
| 4 | `transfer` had only an institution-scope row and no platform fallback. | Seeded a `fixed = 100 XAF` platform default. |
| 5 | `statement_download_*` were missing from `useFeeEstimate.ts` channel map and fallback rates. | Added both mappings with a `{ rate: 0, fixed: 500 }` last-resort fallback. |

## 4. E2E behaviour verified

| Flow | Path | Outcome |
|------|------|---------|
| Admin selects "Statement Download — Consumers App" in Create Fee Structure | `/admin/fee-management` → Structures → Create | Row inserts under platform scope with chosen amount. |
| Statement preview (free) | Consumers/Banking app → Statements | No charge, no `fee_structures` lookup. |
| Statement download (paid) | `statements-generate-pdf` edge function | `resolve_statement_fee` → returns `fee_structures` row; `charge_statement_fee_v2` debits balance with idempotency key; receipt PDF stamped with charged amount. |
| Admin changes fee in Fee Management | UPDATE `fee_structures` row | Next download picks up the new amount on the same request — `STABLE` resolver, no cache. |
| Admin sets `fixed_amount = 0` | Same | `is_enabled = false` returned by resolver → download is free, receipt stamped "Waived". |
| Insufficient balance | Same | `charge_statement_fee_v2` returns `insufficient_funds` → edge returns **402** → UI shows shortfall + "Add funds" CTA. |
| Double-click defence | Same | Same idempotency key returns the original `statement_fee_charges` row; no double-debit. |
