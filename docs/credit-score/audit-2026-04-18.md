# Credit Score System — E2E Audit (v1.0.0)
**Date:** 2026-04-18
**Scope:** Consumer App (`/app/credit`) · Banking App (`/bank/more/credit`) · Business App (`/biz/trust-score`)

---

## 1. System Overview

The CrediQ ecosystem uses three independent scoring engines:

| Engine | Range | Subject | Source of Truth |
|---|---|---|---|
| **CrediQ Personal Score** | 300–850 | Individual user | `credit_profiles` + event-sourced from `credit_events` via `credit-score-engine` |
| **Bank Credit View** | 300–850 | Same user, lender lens (filters customer-only events) | `credit-score-fetch` |
| **Merchant Trust Score** | 0–100 | Business/merchant | `merchant_trust_scores` via `merchant-trust-score` |

## 2. Gap Analysis

| ID | Severity | Area | Gap | Status |
|---|---|---|---|---|
| **G1** | 🔴 Critical | `credit-score-fetch` | Inverted boolean: `useEventSourced = hoursSinceCompute < 24 \|\| !force_refresh` — `force_refresh=true` still returned cached score | ✅ FIXED |
| **G2** | 🟠 High | Business App | No `/biz/trust-score` route — merchants couldn't view their trust score, tier, or breakdown | ✅ FIXED (new page) |

## 3. Implemented Fixes (this audit)

### Fix 1: Force-refresh cache bug (G1)
`supabase/functions/credit-score-fetch/index.ts` — corrected boolean so `force_refresh=true` skips cached profile and triggers re-engine call.

### Fix 2: New `credit-recompute` edge function (G3, G4)
- Authenticated user-triggered recomputation
- 60-second per-user rate limit
- Invokes `credit-score-engine` and returns delta

### Fix 3: Business App Trust Score page (G2)
- New route: `/biz/trust-score`
- Component: `src/pages/business-app/BusinessTrustScore.tsx`
- Shows score (0–100), tier badge (platinum/gold/silver/bronze/unverified), risk level, factor summary, detailed breakdown, history sparkline, improvement tips
- Linked from `BusinessMore` → "Management" section

### Fix 4: Recompute UI (G3)
- **Customer App** (`CustomerCreditScore.tsx`): icon button in header
- **Banking App** (`BankCreditScore.tsx`): "Recompute" pill button next to Back

## 4. Outstanding Recommendations (logged for follow-up)

1. **Lender-only score** (G8): Add a `?lens=lender` parameter to `credit-score-engine` that excludes customer-only event types when banking app fetches.
2. **Loan rehabilitation** (G6): Introduce `LOAN_RECOVERY` event type with positive points after sustained on-time repayment post-default.
3. **Merchant self-recalc** (G9): Allow merchant owners (not just admins) to trigger their own `calculate` action with rate limiting.
4. **Cron schedule** (G10): Add daily cron for `merchant-trust-score?action=batch_calculate`.
5. **Banking placeholder removal** (G7): Hide the `defaultFactors` grid entirely when no real data instead of showing "—".
