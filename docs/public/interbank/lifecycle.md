## Transition Rules

| From | Allowed To |
|---|---|
| created | validated, failed, expired |
| validated | submitted, failed, expired |
| submitted | accepted, rejected, failed, expired |
| accepted | in_process, settled, failed |
| in_process | settled, failed |
| rejected | reversed |
| settled | reversed |
| failed | reversed |
| reversed | (terminal) |
| expired | (terminal) |

## Event-Driven Updates

Every status transition creates an `interbank_status_events` record:
- `payment_id` вҖ” reference to the payment
- `status_from` / `status_to` вҖ” transition
- `source` вҖ” engine, connector, admin, or reconciliation
- `correlation_id` вҖ” for tracing
- `details_json` вҖ” ISO message IDs, reasons, metadata
