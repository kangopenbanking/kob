# Cron Jobs Setup Guide

This document explains how to set up automated cron jobs for continuous system monitoring using pg_cron in your database.

## Overview

Two critical monitoring functions should run on schedules:
- `health-alert-monitor`: Runs every 5 minutes to check system health
- `api-health-collector`: Runs every minute to collect API health metrics

## Prerequisites

Your database needs the following extensions enabled:
```sql
CREATE EXTENSION IF NOT EXISTS pg_cron;
CREATE EXTENSION IF NOT EXISTS pg_net;
```

## Verification

### Check Active Cron Jobs

```sql
SELECT * FROM cron.job;
```

You should see both jobs listed with their schedules.

### Check Execution History

```sql
SELECT * FROM cron.job_run_details
ORDER BY start_time DESC
LIMIT 20;
```

This shows the last 20 executions with their status and any error messages.

## Management

### Unschedule a Job

If you need to stop a cron job:

```sql
SELECT cron.unschedule('health-alert-monitor-every-5-min');
-- or
SELECT cron.unschedule('api-health-collector-every-minute');
```

### Modify Schedule

To change the schedule, unschedule the old job and create a new one with the desired cron expression.

## Cron Expression Format

The cron expressions use the standard format:
```
* * * * *
│ │ │ │ │
│ │ │ │ └─── day of week (0-7, Sunday = 0 or 7)
│ │ │ └───── month (1-12)
│ │ └─────── day of month (1-31)
│ └───────── hour (0-23)
└─────────── minute (0-59)
```

Examples:
- `* * * * *` - Every minute
- `*/5 * * * *` - Every 5 minutes
- `0 * * * *` - Every hour at minute 0
- `0 0 * * *` - Every day at midnight
- `0 0 * * 1` - Every Monday at midnight

## Monitoring Results

Once configured, you can monitor the results:

1. **System Alerts Dashboard**: `/admin/system-alerts` - View real-time alerts generated by health monitoring
2. **API Health Dashboard**: `/admin/api-health` - View API health metrics collected over time
3. **System Monitoring**: `/system-monitoring` - Overall system health overview

## Troubleshooting

### Jobs Not Running

1. Check if pg_cron extension is enabled:
   ```sql
   SELECT * FROM pg_extension WHERE extname = 'pg_cron';
   ```

2. Check cron logs:
   ```sql
   SELECT * FROM cron.job_run_details
   WHERE job_id IN (
     SELECT jobid FROM cron.job WHERE jobname LIKE 'health%'
   )
   ORDER BY start_time DESC;
   ```

3. Verify edge functions are deployed and accessible

### Network Errors

If you see network errors in the cron logs:
- Verify the edge function URLs are correct
- Check that the authorization token is valid
- Ensure pg_net extension is enabled

## Security Notes

- The Authorization token in the cron jobs is the anon key, which is safe to use for scheduled tasks
- Edge functions should validate incoming requests
- Cron jobs run with database permissions, not user permissions
- Sensitive operations should still require proper authentication
